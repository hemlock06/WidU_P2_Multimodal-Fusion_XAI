## §1. 데이터 전략 — 클래스 조건부 조립 채택 (2026-05-30)

**선택**: 모달리티별 실데이터로 인코더를 독립 사전학습하고, 융합 학습용 paired 샘플은
**클래스 조건부 조립(conditional-independence assembly)** 으로 생성. 1단계는 문헌 수치로
캘리브레이션한 사전분포, 2단계는 실데이터 통계로 정합.

**대안**:
- (기각) 동시 레코딩 공개 데이터셋 사용 — 존재하지 않음(아래 조사).
- (보조 채택) 시뮬레이터 — 모달리티 간 상관·동역학 보강용으로 병행(방법 B).
- (2단계 채택) modality-dropout late fusion — 결측 강건성(방법 C).

**이유**: 멀티모달 모델은 인코더 + 융합 헤드로 분리되며, 인코더는 이종 데이터로 독립 학습
가능하나 융합 헤드는 paired 샘플이 필요. 동시 데이터가 없으므로 클래스 조건부 독립 가정하에
조립해 paired 샘플을 합성하는 것이 최선. 실데이터를 재료로 쓰므로 순수 시뮬레이션보다 현실적.

**영향 범위**: 융합 헤드 학습 데이터 전량이 조립/합성. 평가 시 sim-to-real 갭을 명시하고,
2단계 실데이터 정합으로 완화. 모달리티 간 미세 상관(예: 낙상 충격↔놀람 빈맥)은 방법 B로 보강.

### 1.1 공개 데이터셋 조사 결과 (검증됨)

| 데이터셋 | ECG | IMU | SpO2 | 레이블 | 동시? | P2 활용 |
|---|---|---|---|---|---|---|
| PTT-PPG (PhysioNet) | ✅3-lead | ✅가속도+자이로 | ✅ | 활동(정상/걷기/달리기) | ✅ | 클래스 0·1 앵커 |
| SisFall | ❌ | ✅200Hz | ❌ | 낙상15+ADL19 | — | 클래스 3 IMU 앵커 |
| Harespod | ❌ | ❌ | ✅100Hz+HR | 유발 저산소 | — | 클래스 4 SpO2 앵커 |
| SHIMMER | ✅39rec | ✅51Hz | ❌ | 낙상4명+심장이상 | **❌ 별도세션** | **기각** |
| MIMIC-IV-ECG-Ext-ICD | ✅80만 | ❌ | ❌ | ED 응급 ICD | — | P1이 이미 커버 |
| 심혈관(클래스2) | P1(PTB-XL/CPSC) | — | — | — | — | P1 임베딩 직접 |

### 1.2 SHIMMER 기각 사유 (정밀 조사)

처음엔 "ECG+IMU+낙상+심장이상 동시" 후보로 기대했으나, 본문(PMC6920471) 확인 결과:
- ECG와 IMU가 **별도 프로토콜·별도 세션·별도 피험자** → 동시 융합 학습 불가(PTT-PPG와 동일 한계).
- IMU **51Hz** (SisFall 200Hz의 1/4) → 낙상 impact 피크 해상도 부족.
- 낙상은 **4명·매트리스 위 의도적**, ECG는 39레코드·"basic anomalies" 모호.
→ 낙상은 SisFall, 심혈관은 P1이 모든 면에서 우수. SHIMMER 채택 이득 없음.

## §2. 모달리티 정의 — IMU 채택 (2026-05-30)

**선택**: 외부충격(클래스 3) 1차 모달리티를 "자이로" → **IMU(가속도 3축 + 자이로 3축)** 로 변경.

**이유**: 낙상·충격의 1차 신호는 가속도 impact spike(자유낙하 트로프→고가속 피크)이고,
자이로(회전 각속도)는 보조다. 스마트폰은 가속도계·자이로 모두 내장 → 입력 정의 정합.

**영향 범위**: IMU 인코더는 6채널(ax,ay,az,gx,gy,gz) 기반. 핸드크래프트 피처에 SMV·jerk·
tilt(가속도계) + gyro 회전량(자이로) 모두 포함.

## §3. 융합 아키텍처 — conf-routed gate 채택 (2026-05-30, Phase 1)

**선택**: Gated Late Fusion + **confidence-routed gate** (학습 게이트넷 아님).
`gate_w = softmax(conf_m / τ)`, τ=0.15, conf_m = max(softmax(unimodal_logits_m)).

**대안 (모두 기각)**: 학습 gate_net MLP는 3가지 변형(learned/conf-aware/MoE) 모두
**게이트 붕괴**(입력 무관 고정 가중치). 합성셋에선 적응 유인·능력 부재.

**이유**: conf-routed는 학습 파라미터 없어 붕괴 불가, 구조적으로 적응적(샘플별 conf 반영),
투명·해석 가능(P3용). 3시드 확정: clean F1 동률, cardiac·결측 강건성·분산에서 우위.

**영향 범위**: 융합 가중은 confidence(conf)가 단독으로 라우팅을 주도한다(학습 파라미터 없음).

## §4. 전처리 검증 — 통일 IMU 프로토콜 (2026-05-30, 심층 재검토)

**발견된 문제** (재검토):
1. IMU 피처 스케일 불일치: 클래스 1·3(SisFall 보정)과 0·2·4(문헌 추측)가 다른 스케일.
   PTT-PPG sit 실측 jerk_peak=52 vs class0 prior=0.3 (175배).
2. fs·윈도우 불일치: PTT-PPG(500Hz·전체기록) vs SisFall(200Hz·3초). `jerk_peak=Δsmv×fs`는
   fs 의존, `gyro_energy=mean|ω|×지속시간`은 윈도우 길이 의존.

**해결 — 통일 프로토콜** (`calibrate_imu_priors.py`):
- 공통 **200Hz·3초 윈도우**: PTT-PPG를 500→200Hz 리샘플(resample_poly 2/5).
- 5개 클래스 IMU prior 전부 동일 조건 실데이터 재보정:
  - 0 rest ← PTT sit (2855 win), 1 active ← PTT walk+run (5560 win), 3 impact ← SisFall fall (1798)
  - 2 cardiac·4 hypoxia ← PTT sit 기반(저활동) — IMU로는 rest와 구분 불가(설계상 정상,
    심혈관은 ECG·저산소는 SpO2가 구분)
- 효과: jerk_peak rest 52→1.34, 스케일 일관(rest 1.34 < active 9.5 < fall 280).
  모달리티 분리 강화 — IMU-only hypoxia recall 0.70→0.50(SpO2가 유일 구분자로 정상화).
- 재학습 후 test macro-F1=0.954 유지, 게이트 라우팅 클래스별 정상.

## §5. Harespod SpO2 — 절대 스케일 복원 불가, 임상 prior 유지 (2026-05-30)

**발견**: Harespod 공개 SpO2(`spv_5cut.csv`)는 `data_5cut.py`에서 **피험자별 min-max 정규화**
(`(x-min)/(max-min)`). min/max가 자르기 전 전체 기록 기준 + SpO2=0 dropout 아티팩트 포함.
→ 피험자마다 cut 구간 범위 제각각([0,1], [0.028,0.492]). **절대 SpO2% 복원 불가**.

**결정**: SpO2 prior(`SPO2_PRIORS`)는 **임상 문헌값 유지**(nadir≤88%, desat≥4%, time_below_90 등
표준 임상 정의). Harespod는 유발 저산소 desaturation 패턴의 **정성적 확인**으로만 활용.

**근거**: 임상 SpO2 임계값은 표준이고, SpO2-only 분류기가 hypoxia recall 0.94 달성 — 충분.
정직성 위해 합성-실 갭을 본 기록에 명시.

**추가 (2026-05-30): Harespod 고도-SpO2 재앵커링 검증**
논문(Sci. Data 2024, DOI: 10.1038/s41597-024-03065-x) 확인: 레벨 20/25/30/35/40 =
2.0/2.5/3.0/3.5/4.0km 고도 (저압산소챔버). 앵커(2.0km→95%, 4.0km→85%)로 역산:
- spo2_nadir prior 84% ≈ 역산 4km 평균 85.0% ✓
- spo2_mean prior 89% ≈ 역산 3.5km 88.3% ✓
- spo2_std prior 2.8%p ≈ 역산 4km 2.45%p ✓
**한계**: 앵커가 문헌값 → 순환적. 독립 측정 아닌 수렴 검증. 응급 임계(<80%) 미검증.

## §6. 정직성 감사 및 누출 교정 (2026-05-30)

### 6.1 발견된 결함 3건

**결함 1 (치명적): ECG 임베딩 train/test 누출**
P1Cache가 train+val 풀에서 복원추출 후 70/15/15 분할
→ test 임베딩 90.5%가 train과 동일 → 성능 부풀림 (F1 0.93→0.96).

**수정**: `P1Cache(split="train_val")` vs `P1Cache(split="test")`로 분리.
- P2-train/val: P1 train+val 풀 (5290 레코드)
- P2-test: P1 test 풀 (936 레코드, 신규)
- ECG 누출 0% 확인 후 `p2_synth_vf_{train,val,test}.npz` 생성

**결함 2: Bootstrap IMU 근접 누출**
실벡터 ~1800개를 5% jitter로 재사용 → test가 train의 근접복제.
bootstrap의 F1 우위(0.962)가 누출 아티팩트로 판명 → **MVN으로 정정**.

**결함 3: raw 768 임베딩 과적합**
vf 재학습: val 0.963, test 0.855, 격차 0.104.
비심장 클래스(rest/active/impact/hypoxia)가 0.81~0.87로 하락.
원인: NSR 768차원이 train pool(776개) 특이 방향을 외웠다가 test pool(130개)에서 실패.

### 6.2 병목 임베딩으로 해결

**실험**: `emb_bottleneck`: 0 (제거), 16, 32 (3시드 비교)

| 설정 | test F1 | val↔test 격차 | cardiac | 비심장 평균 |
|---|---|---|---|---|
| full 768→256→128 | 0.859 | 0.104 | 0.935 | 0.83 (과적합) |
| no_emb (0차원) | 0.929 | 0.004 | 0.887 (-0.048) | 0.94 |
| **bn=16 (768→16→128)** | **0.941** | 0.004 | **0.946** | 0.94 |
| **bn=32 (768→32→128)** | **0.942** | 0.007 | **0.948** | 0.94 |

**결론**: 병목이 cardiac(-0.048 희생 없이)과 일반화 격차 해소를 동시 달성.
cardiac 신호는 본질적으로 저차원(정상 vs 비정상 형태)이라 16~32차원이면 충분.
NSR은 768차원이어야 외울 수 있어 좁은 병목으로 자연 차단.

**아키텍처적 의미**: "P2는 P1 raw 768 표현이 아니라 압축된 cardiac 신호만 필요"
= 원래 모달리티 책임 분리 설계의 정제된 형태. 데이터로 증명됨.

### 6.3 정직한 최종 성능

**test macro-F1 ≈ 0.941~0.942** (부풀려진 0.96에서 교정됨).
이 수치는 누출 0%, 실 공분산(MVN), 병목 임베딩으로 일반화된 진짜 값.

**근본 한계 (변하지 않음)**: 합성셋 위 성능. 실 응급 멀티모달 데이터 부재.
실세계 응급 탐지 성능이 아닌 "아키텍처 설계 타당성 검증" 수준으로 명시.

### 6.4 포트폴리오 서사

"내 파이프라인을 스스로 누출·과적합 검사해 3개 결함 발견·교정,
닫힌 val↔test 격차(0.104→0.007)로 정직한 일반화 성능 보고."
0.962보다 0.942가 리뷰어에게 더 신뢰받을 수 있는 이유.

## §7. Cross-Modal Attention 전환 결정 (2026-05-30)

### 7.1 conf-routed의 구조적 한계 — 오경보 억제 불가

conf-routed gate = "가장 확신하는 single expert에게 가중치 몰아주기".
오경보(false positive) 시나리오에서 구조적 실패:

| 상황 | single expert | 정답 | 문제 |
|---|---|---|---|
| 운동 중 잠깐 낙상 | IMU conf 0.97 → 낙상 응급 | 정상 (운동 재개) | IMU 단독 이겨버림 |
| 만성 비정상 ECG | ECG conf 0.9 → 심혈관 응급 | 개인 정상 베이스라인 | ECG 단독 이겨버림 |
| 수면무호흡 | SpO2 conf 0.95 → 저산소 응급 | 정상 수면 | SpO2 단독 이겨버림 |

→ 이 케이스들은 **joint-판별 구조**: 단독 지표가 강하게 오답을 가리키고, **결합으로만 정답**.
→ conf-routed는 이 케이스에서 오경보를 **증폭**함.

### 7.2 "F1으로 어텐션 정당화는 논리 안 맞다" — 사용자 지적 수용

조건부 독립 합성 데이터에서 어텐션 테스트 → F1 안 오르는 게 당연(데이터에 joint 구조 없음).
올바른 판정 기준: **오경보율(FP rate on confounders)** — 임상적으로도 옳은 지표.

### 7.3 결정: Cross-Modal Attention + Confounder 데이터

**채택 근거**:
- P1 점수(emergency_score, cardiac_probs)가 어텐션 토큰에 실려 실제 소비됨
- "ECG는 응급인데 IMU는 활발" → 운동 중 낙상 판별 가능 구조
- attention_weights → P3 XAI ("어떤 모달리티가 이 판정에 주목했나")

**필요한 새 데이터 (합성)**:
- transient_fall: IMU impact 큰데 직후 활동 재개 → 정상
- chronic_ecg: ECG 비정상인데 IMU/SpO2 정상 → 정상
- sleep_apnea: SpO2 주기적 하강 + IMU 정지(수면) → 정상

**평가 지표 재정의**:
- 기존: macro-F1 (5-class 분류 정확도)
- 추가: **confounder FP율** (단일 지표 강해도 맥락상 정상인 케이스 오경보율)
- conf-routed가 FP 높고, cross-attn이 낮으면 → 어텐션 정당화 완성

**conf-routed (bn=16, test F1=0.938~0.942)는 ablation 베이스라인으로 유지.**
